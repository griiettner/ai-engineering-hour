Módulo 10: Memória com Busca Local (RAG)

Memória Inteligente

📉 Otimizando a Janela de Contexto

Carregar todo o repositório em cada prompt é caro e confuso para a IA. A memória com busca local (RAG - Retrieval-Augmented Generation) permite que a IA busque apenas o parágrafo exato que precisa.

Sem Busca (Full Load)

O contexto cresce linearmente com o número de documentos, estourando a janela rapidamente.

Com Busca Local (RAG)

Apenas snippets relevantes são carregados. O consumo de tokens permanece estável e preciso.

📂 O que é qmd / Memória Consultável?

Inspirado em ferramentas como o qmd, o conceito é simples: transformar seus arquivos Markdown em uma base de dados que a IA pode "perguntar" antes de responder.

📝 Markdown como DB

Seus arquivos de documentação (ADRs, specs, guides) são segmentados em blocos lógicos que a IA consegue identificar individualmente.

  • • Tags de busca rápida (Keywords)
  • • Metadados por arquivo
  • • Links entre conceitos

🔍 Retrieval Automático

Ao invés de você anexar os arquivos, a IA roda um comando interno de busca (como o grep ou busca vetorial local) para encontrar o contexto.

$ qmd search "auth flow"

A Grande Vantagem:

"Você não precisa que a IA saiba tudo o tempo todo. Você só precisa que ela saiba ONDE buscar quando for necessário."
