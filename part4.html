📉 A Ponte entre "Achar" e "Saber"

O maior medo de um dev é a IA alucinar um nome de tabela ou um endpoint que não existe. O MCP (Model Context Protocol) reduz drasticamente isso ao dar "olhos" para a IA acessar seus sistemas internos em tempo real em vez de depender apenas do treinamento estático.

Sem MCP, Conhecimento Estático

A IA usa apenas o que aprendeu no treinamento (com data de corte). Se seu banco mudou ontem, ela vai sugerir código baseado em um schema que não existe mais.

Com MCP, Contexto ao Vivo

A IA consulta a Fonte da Verdade (banco real, logs ao vivo, tickets abertos) antes de responder. O erro de contexto cai para próximo de zero nos dados conectados.

Importante

MCP não elimina alucinações em raciocínio, ele elimina erros de contexto factual sobre os seus sistemas.

🧩 Ferramentas Conectadas

O MCP funciona através de "servidores". Cada servidor conecta a IA a uma ferramenta específica. Clique nas categorias para ver o que a IA passa a "enxergar" no seu projeto.

🗄️

Bancos de Dados

PostgreSQL, MySQL, Redis

A IA lê o schema real, analisa índices e valida queries antes de sugerir código SQL.

📋

Gestão & Documentação

Jira, Linear, Confluence, Notion

A IA entende o contexto do ticket atual e consulta documentações internas sem você precisar copiar nada.

📊

Observabilidade

Sentry, CloudWatch, Datadog

Investigue bugs em tempo real. A IA correlaciona o erro do log com a linha exata do código.

🌐

Documentação Viva

Fetch, MDN, docs oficiais

A IA consulta documentação atualizada e detecta breaking changes que não estavam no seu treinamento original.

🏗️ Como o MCP Conecta?

O MCP usa um modelo Cliente-Servidor. O Claude Code (cliente) se comunica com servidores locais ou remotos que "falam a língua" das suas ferramentas via JSON-RPC.

📦 Transporte stdio (local)

O Claude Code inicia o servidor como um processo filho. Comunicação via stdin/stdout. Mais simples, sem rede, ideal para ferramentas locais como banco de dados de desenvolvimento.

🌐 Transporte HTTP + SSE (remoto)

Servidor roda de forma independente (pode ser na nuvem). A IA se conecta via URL. Ideal para serviços compartilhados como Jira corporativo ou APIs internas de equipe.

⚡ Debugging em Tempo Real

Veja como a IA resolve um problema complexo quando tem acesso ao MCP de Logs e Banco de Dados simultaneamente. Clique em "Próximo Passo" para avançar a investigação.

Dev:

"Tivemos um erro 500 no checkout há 10 minutos. Investigue usando o Sentry e o banco."

› sentry.get_issues(project="checkout", last_minutes=15)

Encontrei 3 ocorrências de DatabaseError: Unique constraint failed on orders.idempotency_key entre 14:22 e 14:31. Todos no endpoint POST /v2/checkout.

🤖

Claude (via MCP PostgreSQL):

› postgres.query("SELECT constraint_name FROM information_schema.table_constraints WHERE table_name='orders'")

O campo idempotency_key tem constraint UNIQUE mas o código não faz upsert, ele tenta INSERT direto. Isso falha em retentativas do cliente.

🤖

Claude (diagnóstico + fix):

Root cause: a migration da semana passada adicionou a constraint mas o service não foi atualizado. Solução: usar INSERT ... ON CONFLICT DO NOTHING. Quero gerar o patch agora?

+ INSERT INTO orders (...) VALUES (...)
+ ON CONFLICT (idempotency_key) DO NOTHING;

⚠️ Limites & Cuidados

MCP é poderoso e isso também significa que os riscos são reais. Antes de conectar a IA aos seus sistemas, entenda os limites.

🔴 Riscos críticos

🔓

Acesso direto a produção

Se você conectar um MCP server com credenciais de produção, a IA pode executar queries destrutivas se instruída ou se um hook mal configurado disparar um comando errado.

❌ Credenciais de produção no MCP local
✅ Credenciais de staging/read-only para desenvolvimento
🧩

Qualidade dos servidores da comunidade

A maioria dos servidores MCP disponíveis são projetos open source em estágio inicial. Alguns podem ter bugs, comportamento imprevisível ou expor mais dados do que o necessário. Sempre revise o código antes de usar em contexto corporativo.

✓ Prefira servidores oficiais (@modelcontextprotocol/*) ou auditados pelo time.

🟡 Pontos de atenção

💸

Custo de tokens explode com ferramentas

Cada chamada de ferramenta MCP consome tokens tanto para o input (resultados do servidor) quanto para o raciocínio. Um schema de banco com 80 tabelas injetado no contexto pode custar caro em longas sessões.

✓ Prefira servidores que retornam resultados filtrados/paginados, não dumps completos.

🤖

MCP não resolve problemas de raciocínio

MCP fornece contexto factual o que existe no sistema. Mas a IA ainda pode interpretar esse contexto errado ou tomar uma decisão arquitetural ruim com base em dados corretos. O julgamento de o que fazer com a informação ainda é humano.

🔒

Dados sensíveis no contexto da IA

Ao conectar o Sentry ou banco, dados de usuários reais podem entrar no contexto da conversa. Verifique a política de privacidade do provedor de IA e as regulamentações (LGPD) antes de conectar dados PII.

✓ Use dados anonimizados no ambiente de desenvolvimento. Nunca conecte PII em prod sem revisão legal.

🚀 Como começar com MCP hoje

Quatro passos para sair do zero e ter sua primeira ferramenta conectada funcionando em uma tarde.

1

Comece com o MCP de Fetch (zero risco)

O servidor @modelcontextprotocol/server-fetch não acessa sistemas internos, apenas faz requisições HTTP. Perfeito para começar sem risco.

# Adicionar em .claude/settings.json
"mcpServers": {
  "fetch": {
    "command": "npx",
    "args": ["-y", "@modelcontextprotocol/server-fetch"]
  }
}
2

Conecte o banco de dados de staging (read-only)

Use o PostgreSQL MCP apontando para staging com um usuário de leitura. Teste pedindo para a IA listar as tabelas e descrever o schema.

💬 "Use o MCP do banco para me descrever o schema da tabela de orders e identificar possíveis problemas de performance."
3

Combine MCP + Plan.md para uma feature real

Peça para a IA criar um Plan.md consultando o banco real antes de propor mudanças. O plano agora inclui nomes de tabelas e campos reais, não supostos.

💬 "Antes de criar o plan.md, consulte o schema via MCP e liste os campos exatos que precisarão ser alterados na tabela de usuários."
4

Defina uma política de MCP para o time

Documente no CLAUDE.md quais MCP servers estão disponíveis, quais são read-only e em quais contextos cada um deve ser usado.

# Adicionar no CLAUDE.md

## MCP Servers disponíveis
- `postgres`: banco de staging, READ-ONLY. Use para validar schemas.
- `sentry`: erros de produção dos últimos 7 dias.
- `fetch`: para consultar documentação externa.

⚠️ Nunca use credenciais de produção localmente.
